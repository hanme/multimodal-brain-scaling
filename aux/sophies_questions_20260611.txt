Overview:
Search aimed to identify public EEG datasets that contained audio/audio-visual stimuli, EEG recordings, and metadata to map events to EEG
To identify datasets I manually reviewed Zenodo, OpenNeuro, OSF, PhysioNet, Dryad, Google Search, Nature Scientific Data, and some public lists, as well as performed LLM-assisted search 
Inclusion criteria:
EEG was recorded while subjects listened to audio/audio-visual stimuli passively
The original audio stimuli are available, either as .wav, etc. or through manual creation via a description and a presentation software
EEG responses are available as BrainVision or .mat
Metadata exists to link align EEG response to corresponding stimulus at a fine-grained level
Stimuli must not include spatial/synthetic manipulation, competing sound streams, different sounds to each ear, attention switching instructions.
Datasets collected using instructions not involving attention, such as imagery or imagination, were acceptable when needed
Preferred stimuli were naturalistic, such as speech, music, tones, natural sounds and did not include a visual component
Main findings (addressing your criteria):
Naturalistic stimuli: are available as speech, music, natural sounds (e.g. dog bark), and ERP paradigms (e.g oddball)
Auditory vs. audio-visual: Since there are several high-quality datasets for audio-only stimuli, I did not include audio-visual stimuli

Proposed Plan for Creation of EEG Encoding Models 
Models list: Whisper-tiny, base, small, medium, large; AST, Wav2vec2 base, large, VGGish
1. Select dataset(s) from dataset list and pre-process/segment the datasets
Preprocess the EEGs using EEG standardization
Use metadata to segment EEG and audio such that stimuli (e.g. words) are not cut across segments, such that the segments are under the temporal bin size for the model 
Two segmentations: target segment size of 10’’ (non-whisper), 30’’ (whisper)
2.  Create audio + EEG set of pairs:
(Audio sample A_i, Corresponding EEG Segment Y_i) for each original Audio stimuli A, EEG sample Y, where i = time index
E.g. a 2 min Audio / EEG recording may be split into 12 x ~10’’ segments and 4 x 30’’ segments
3. For each model in the model list, retrieve final embedding X_i for each audio stimuli sample A_i creating the dataset of pairs: (X_i, Y_i) 
4. For each model in the models list above, train a separate ridge regression model M to predict Y_i from X_i (results in 9 models)
5. Evaluate the encoding performance for each model M on a held-out set of the stimuli (~10%)
Then proceed with process as before to select electrodes as the new “MMN Unit Set”:
For each model in the models list:
For each audio stimuli pair A = (A_std, A_dev) in the MMN literature set:
Retrieve the predicting EEG Y across all electrodes using the linear mapping model M established above for the given model
Then the difference of the EEGs can be computed D_A = Y_dev-Y_std where D_A is matrix of dimension electrodes x time
For each electrode, compute its response r to a stimuli by using the baseline normalized negative peak metric on D_A discussed before, producing a vector of all the responses of each electrode with the dimension # of electrodes
Join the vectors for each electrode to get a matrix of dim electrodes  x number of stimuli
each row represents the electrodes responses r to all the literature stimuli
Perform a 1–sided 1 sample t-test with null hypothesis of mean(r) = 0 on each row of the matrix to obtain a corresponding p-value for each electrode
Perform multiple testing correction on all p-values (number of tests = number of electrodes)
This is probably not even necessary now since before had thousands of units 
Take the top 5% of electrodes as MMN set and/or set a threshold for p-value to be included
Would be ideal if these electrodes are the MMN electrodes in human studies!

Questions for Kadir - Jun 10, 2026
EEG Standardization: We have been investigating the effect of increasing training set size for the mapping function performance between model features and EEG, by combining multiple datasets (D1+D2 → D3). D1 and D2 differ in # channels/subjects/stimuli.
Initially, D2's EEG had already been normalized to near-0 scale, while D1's EEG was on a raw-amplitude scale. When concatenated for D3, this scale mismatch created a confound rather than increase in encoding performance. 
Model essentially trivially predict "which dataset is this trial from" from the target values themselves, grossly inflating D3's correlations without any corresponding change in the noise ceiling.
I’ve attempted to fix this by per-channel z-scoring both D1's and D2's EEG using each dataset's own training-split mean/std before concatenation (doesn't change the noise ceiling or each dataset's standalone Pearson r). Result:
D3's overall statistics are back in the normal range (similar medians to D1/D2 alone), so there's no conclusive "more data = better encoding" effect
On the curated high-noise-ceiling electrode set, the not much more promising:
 Fz appeared to improve with D3 (effect size 0.53→0.59 for whisper-base, 0.49→0.63 for whisper-small).
However, a follow-up check (scoring the D3 RR model separately on each dataset's own test set) showed this was partly attributed to a pooled-test-set: when scored within-dataset only, Fz's "improvement" disappears (D3 ≈0.50, same as D1-alone =0.49–0.53).. 
One electrode (TP7) showed a large new effect (0.60) not present in either dataset alone, only D3. But when conducting the same test as above, TP7's effect is reduced (~0.33, not 0.60).
So combining datasets did not produce significant improvement in encoding performance yet.
I am unsure how to properly standardize EEG datasets and enable pooling of data across data sources.
For multi-source datasets do you have recommendations for: feature/EEG standardization? Test set construction? for computing noise ceiling corrected values?

2. Noise correction:
For our dataset, there are no repeated trials per subject. Right now, I compute NC by splitting the 19 subjects into two groups, averaging EEG within each group, and correlating the two group-level averages (ie. cross-subject reliability), then applying the Spearman-Brown + sqrt(NC/100) correction as was done originally.
For this paper, is  the NC within-subject (repeated presentations) or also across-subject?
If within-subject (I assume this) we'd expect our cross-subject NC to be a more conservative ceiling than within-subject NC in general, which would inflate our NC-corrected scores relative to the visual benchmarks' scale.
Is the across-subject implementation an appropriate moderating factor for noisy data rather than within-subject? Would you recommend a different approach for noise correction in datasets with 1-rep-per-stimuli-per-subject?
If the above is too complex, would it be possible to use non-corrected r values?
